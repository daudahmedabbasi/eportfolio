Object-oriented programming moved developers away from the bits and bytes on which computers operate, towards the concept of real-life objects which can be more realistically visualised, allowing abstraction of the detail and ability to concentrate on the function(s) required of the application. Object-oriented software has evolved among other incarnations of programming language, which include procedural code, component-based, and service-oriented software.

Unified Modelling Language

UML is an approach to create diagrams using a recognised format to document aspects associated with a software system. UML, the Unified Modelling Language, is a technique which can support the software analysis, design and development process, with different UML models being applicable to different stages of the development process. The UML models produced can be used to communicate the proposed software with stakeholders, ensuring that everyone has a consistent perspective of the proposal. A common syntax is used in UML models, which supports the ease of understanding between stakeholders.

UML approach can help provide a useful high-level overview of a system that can be used to check whether the system's design meets the agreed specification.

Polymorphism

Polymorphism is the situation when a child class can assume the form of any class in the parent hierarchy. Methods of the child class can have the same name as methods of the parent class, yet perform different functionality.

Dynamic Polymorphism: With dynamic polymorphism, or method overriding, methods within a class hierarchy can have the same method signature. When an object's method is called, and the object has been assigned to a class reference, the method in the object's class is executed, and not the method in the parent class.
