Project: Autonomous Vehicle Software Architecture

Self-driving cars are vehicles that can sense their surroundings and move without human intervention. Autonomous vehicles collect real-time data from systems such as GPS, RADAR, LIDAR, sensors, cameras and require software to manage and manage the data collected by the systems. so that the vehicle can be driven without human intervention.
This software module performs three main functions: visualization of the environment. environmental mapping and movement planning

Visualization of the environment

This module has two main components: Find the vehicle, recognize and recognize the main components of the environment. The device uses multiple data streams, including current GPS location, inertial measurement unit (IMU) measurements, and vehicle odometer to connect them to determine the exact location of the car. In some localization models, LiDAR and camera data are also combined to increase accuracy. In general, object classification and identification is divided into two parts. First, it detects moving objects in the environment. Then identify static objects in the environment. The tracking model does not only provide the current location of the moving object. but also provides a history of the trails in the environment. The path history is used together with future path prediction graphs for all smart objects.

Environmental mapping

Environmental maps show different environments around driverless vehicles. There are three types of components used by this module. Residential Grid Maps, Site Maps and Detailed Road Map. A grid map shows all stationary objects around the vehicle, LiDAR are mainly used to collect data for grid mapping. Several filters were first applied to LiDAR data to make it useful for mapping. This way we can remove the uncertainty from the measurement data and improve the maps over time. Location maps created with LiDAR or camera data are used in the location model to improve vehicle location estimates. The sensor data is compared to this map while driving to identify the vehicle's movement relative to the local map. This motion is combined with other collected data essential for determining the location of the car. The detailed driving map is a multi-part map that shows the current driving conditions of autonomous vehicles, collect road signs and traffic signs that you can use to plan your trip. These maps are a combination of pre-stored information about the current static environment and information gathered from future static observations. Model mapping and environmental awareness together improve the efficiency of both models, for example the concept model provides the static environmental data needed to update a detailed route plan. Then predictive models are used to accurately predict moving objects.

Motion Planner

Defining the movement of autonomous vehicles is a complex task that must be solved as part of the integration process. on the contrary, most self-driving cars today can avoid these problems. Mission planners handle long-term planning from start to finish. The mission planner follows the starting point to the end of the route and sends it to the next part. short-term planning behaviour planner. Behaviour planners are responsible for finding and following the right path.

#Import pygmae and initiate it
import pygame
pygame.init()
#load the display window, track, car and decide the starting point of car on track
window = pygame.display.set_mode((1200, 400))
road = pygame.image.load('road.png')
car = pygame.image.load('car.png')
car = pygame.transform.scale(car, (30, 60))
#define variable for car position, camera position, camera viewing point, clock and direction
car_position_x = 155
car_position_y = 300
Camera_view_dist = 25
camera_x_position = 0
camera_y_position = 0
direction = 'up'
clock = pygame.time.Clock()
Drive = True
while Drive:
#define the quit event
for event in pygame.event.get():
if event.type == pygame.QUIT:
Drive = False
#timer to dcide the speed of event(car speed)
clock.tick(60)
#define the camera position on car
camera_x = car_position_x + camera_x_position + 15
camera_y = car_position_y + camera_y_position + 15
#Camera input
up_motion_pix = window.get_at((camera_x, camera_y - Camera_view_dist))[0]
right_motion_pix = window.get_at((camera_x + Camera_view_dist, camera_y))[0]
down_motion_pix = window.get_at((camera_x , camera_y + Camera_view_dist))[0]
print(up_motion_pix, right_motion_pix)
#change direction (take turn)
if direction == 'up' and up_motion_pix != 255 and right_motion_pix == 255:
direction = 'right'
camera_x_position = 30
car = pygame.transform.rotate(car, -90)
assert right_motion_pix == 255
elif direction == 'right' and right_motion_pix != 255 and down_motion_pix == 255:
direction = 'down'
car_position_x = car_position_x + 30
camera_x_position = 0
camera_y_position = 30
car = pygame.transform.rotate(car, -90)
assert down_motion_pix == 255
elif direction == 'down' and down_motion_pix != 255 and right_motion_pix == 255:
direction = 'right'
car_position_y = car_position_y + 30
camera_x_position = 30
camera_y_position = 0
car = pygame.transform.rotate(car, 90)
